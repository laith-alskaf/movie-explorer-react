# 🎓 الدليل البرمجي الشامل لمفاهيم React.js النظيفة والمتقدمة

مرحباً بك يا زميلي المهندس في هذا الدليل التعليمي المصمم خصيصاً لمساعدتك على فهم عميق للتقنيات والأنماط المعمارية (Architectural Patterns) المستخدمة في هذا المشروع بعد إعادة هيكلته.

سنتناول هنا المفاهيم البرمجية الأساسية والمتقدمة، كيف تعمل تحت الغطاء، متى تستخدمها، ما هي البدائل المتاحة، والفرق الجوهري بينها وبين الطرق التقليدية لمساعدتك في اتخاذ أفضل القرارات المعمارية وتوجيه الـ AI Agent بثقة واحترافية.

---

## 1. طبقة الخدمات (Services Layer) مقابل الاتصال المباشر (Inline API Calls)

### ❌ الطريقة القديمة (قبل التحسين)
كان الكود يقوم بطلب البيانات مباشرة داخل المكون الرئيسي باستخدام دالة `fetch`:
```javascript
const response = await fetch(`${API_BASE_URL}/discover/movie...`, API_OPTIONS);
```

### ✅ الطريقة الاحترافية (بعد التحسين)
فصل المنطق في ملف مستقل كخدمة مخصصة:
```javascript
// في tmdb.service.js
const tmdbService = {
  fetchPopularMovies: async () => { ... }
}
```

### 🧠 الشرح والمقارنة:
*   **ما هو هذا المفهوم؟** فصل منطق الاتصال بالسيرفر وجلب البيانات (Data Fetching Logic) تماماً عن طبقة العرض (UI Layer).
*   **متى يُستخدم؟** في كافة المشاريع الحقيقية والمهنية دون استثناء.
*   **لماذا هو أفضل شيء؟**
    1.  **مبدأ المسؤولية الفردية (Single Responsibility):** المكون مسؤول فقط عن عرض البيانات وتصميم الواجهة، وليس له علاقة بكيفية وتفاصيل جلب هذه البيانات من الويب.
    2.  **سهولة الصيانة والتعديل (Maintainability):** إذا أردت استبدال مكتبة `fetch` بمكتبة `Axios`، أو إذا قامت شركة الأفلام بتغيير روابط الـ API الخاصة بها، ستقوم بالتعديل في ملف واحد فقط (`tmdb.service.js`) دون الحاجة لتعديل أو لمس أي مكون واجهة مستخدم.
    3.  **إعادة الاستخدام (Reusability):** يمكنك استخدام نفس دالة جلب البيانات في عدة صفحات مختلفة بسهولة دون تكرار كود الاتصال.

## 2. الهوكس المخصصة (Custom Hooks) مقابل المنطق الداخلي (Inline Component Logic)

### ❌ الطريقة القديمة
كتابة حالات التطبيق (`useState`) وتأثيرات التشغيل (`useEffect`) لجلب البيانات مباشرة داخل جسم المكون:
```javascript
const App = () => {
  const [movies, setMovies] = useState([]);
  useEffect(() => { fetchMovies() }, []);
  // ... يعرض الواجهة
}
```

### ✅ الطريقة الاحترافية
تجريد المنطق بالكامل داخل هوك مخصص:
```javascript
const HomePage = () => {
  const { movies, isLoading } = useMovies(searchTerm);
  // ... يعرض الواجهة مباشرة
}
```

### 🧠 الشرح والمقارنة:
*   **ما هو الـ Custom Hook؟** هو دالة جافا سكريبت عادية يبدأ اسمها بـ `use` ويمكنها استدعاء هوكس React الأخرى مثل `useState` و `useEffect`.
*   **لماذا نستخدمه؟** لحفظ المنطق التفاعلي (Stateful Logic) وتجريده. المكونات تصبح نظيفة جداً وخالية من التعقيد الرياضي أو البرمجي وتركز فقط على الـ JSX.
*   **ما الفرق والبدائل؟**
    *   **البديل 1: كتابة الكود داخل المكون.** (مقبول فقط في التطبيقات البسيطة جداً "Todo List"، لكنه يسبب فوضى عارمة وتداخل للمسؤوليات في المشاريع الكبيرة).
    *   **البديل 2: استخدام مكتبات إدارة الكاش مثل React Query (TanStack Query).** وهو الخيار الأفضل إطلاقاً للمشاريع الضخمة التي تعتمد على بيانات خارجية متغيرة باستمرار، لأنها توفر كاش مدمج (Caching) وتحديث تلقائي للبيانات في الخلفية دون كتابة `useState` أو `useEffect` يدوياً.

## 3. التوجيه العميل (Client-Side Routing) مقابل التوجيه التقليدي (Server-Side Routing)

### ❌ الاستخدام التقليدي للروابط
```html
<a href="/movie/123">مشاهدة التفاصيل</a>
```

### ✅ استخدام React Router DOM
```javascript
import { Link } from 'react-router-dom';
<Link to={`/movie/${id}`}>مشاهدة التفاصيل</Link>
```

### 🧠 الشرح والمقارنة:
*   **ما الفرق البرمجي؟**
    *   عند استخدام وسم `<a>` التقليدي، يقوم المتصفح بطلب الصفحة بالكامل من السيرفر، مما يسبب **إعادة تحميل كاملة للموقع (Full Page Reload)** وظهور شاشة بيضاء مؤقتة وضياع حالة التطبيق (State).
    *   عند استخدام `<Link>` من مكتبة `react-router-dom`، يتم اعتراض الحدث برمجياً ويقوم جافا سكريبت بتحديث الجزء المطلوب من الشاشة فقط وتحديث رابط الـ URL في المتصفح تلقائياً (HTML5 History API) بسرعة فائقة ودون أي إعادة تحميل للموقع. هذا هو أساس تطبيقات الصفحة الواحدة (Single Page Applications - SPA).
*   **متى نستخدم useParams و useNavigate؟**
    *   `useParams()`: تستخدم للحصول على القيم الديناميكية من الرابط. مثلاً إذا كان الرابط `/movie/:id` ودخل المستخدم إلى `/movie/550` فإن `useParams()` ستعطيك `{ id: '550' }` لكي تعرف أي فيلم يجب جلب تفاصيله.
    *   `useNavigate()`: تستخدم للانتقال برمجياً بين الصفحات داخل الكود (مثلاً: الانتقال للرئيسية فور نجاح تسجيل الدخول، أو استخدام `navigate(-1)` للرجوع خطوة للخلف مثل زر المتصفح).

## 4. التحكم بمعدل الطلبات (Debouncing)

### ❌ المشكلة بدون Debounce
في نظام البحث، كل حرف يكتبه المستخدم يسبب تشغيل دالة البحث مباشرة:
*   المستخدم يكتب "matrix" (خمسة أحرف).
*   يتم إرسال 5 طلبات متتالية للـ API:
    1. `m`
    2. `ma`
    3. `mat`
    4. `matr`
    5. `matri`
    6. `matrix`
    هذا يسبب بطء شديد في واجهة المستخدم وقد يعرضك للحظر (Rate Limiting) من سيرفر البيانات.

### ✅ الحل باستخدام `useDebounce`
يتم تأخير تحديث النص الفعلي الممرر للبحث بمقدار 500 مللي ثانية مثلاً. طالما أن المستخدم مستمر في الكتابة ولم يتوقف لأكثر من نصف ثانية، فلن يتم إرسال أي طلب. بمجرد توقفه، يتم إرسال طلب واحد نهائي بالكلمة الكاملة.

## 5. العرض الشرطي الآمن (Safe Conditional Rendering)

### ❌ الوقوع في فخ الـ Crash أو الأرقام الوهمية
في React، نستخدم المعامل الشرطي `&&` لعرض المكونات شرطياً:
```javascript
{trendingMovies.length > 0 && <TrendingList />}
```
**الخطر المحدق:** 
1. إذا كانت قيمة `trendingMovies` هي `undefined` أو `null` (مثل المشكلة التي قمنا بحلها)، سيحدث انهيار كامل للتطبيق (Crash) لأن المتصفح لا يستطيع قراءة خاصية `length` من كائن غير موجود.
2. إذا كانت القيمة رقماً صفرياً `0` (مثلاً عدد عناصر السلة هو 0)، فإن React سترندر الرقم `0` مباشرة على الشاشة بصرياً بدلاً من عدم عرض أي شيء، لأن الرقم صفر يعتبر قيمة معتبرة في جافا سكريبت عند التقييم بالمنطق `&&`.

### ✅ الطريقة الاحترافية والأمنة إطلاقاً
استخدام الفحص المزدوج أو التقييم الصريح لنوع البيانات:
```javascript
// الطريقة الأكثر أماناً
{trendingMovies && trendingMovies.length > 0 && <TrendingList />}

// أو استخدام معامل التحويل البولياني الصريح
{Boolean(moviesCount) && <CartIcon />}
```

## 6. تفكيك الخصائص (Props Destructuring)

### ❌ الكتابة التقليدية
```javascript
const MovieCard = (props) => {
  return <h3>{props.movie.title}</h3>
}
```

### ✅ التفكيك المباشر (Destructuring)
```javascript
const MovieCard = ({ movie: { title, release_date } }) => {
  return <h3>{title}</h3>
}
```

### 🧠 الشرح والمقارنة:
*   **ما هو؟** ميزة من ميزات جافا سكريبت الحديثة (ES6) تسمح لنا باستخراج القيم من الكائنات وتعيينها لمتغيرات مباشرة.
*   **لماذا نستخدمه؟** يجعل الكود أقصر بكثير وأسهل للقراءة ويغنيك عن تكرار كلمة `props.` أو `movie.` عشرات المرات داخل المكون. كما أنه يوضح فوراً من سطر البارامترات الأول ما هي البيانات الدقيقة التي يحتاجها هذا المكون ليعمل.

## 🚀 ملخص المبادئ المعمارية الاحترافية (SOLID & Clean Code)

لكي تتميز كمطور محترف وتوجه الذكاء الاصطناعي بشكل سليم، ضع هذه القواعد نصب عينيك دائماً:

1.  **مبدأ المسؤولية الفردية (SRP):** اسأل نفسك دائماً: "هل هذا الملف يقوم بوظيفة واحدة فقط؟" إذا كان المكون يعرض البيانات ويجلبها في نفس الوقت، اطلب من الـ AI فصل جلب البيانات في هوك مستقل.
2.  **مبدأ دون تكرار (DRY - Don't Repeat Yourself):** إذا وجدت نفس التنسيق أو نفس كود الـ HTML مكرر في مكانين أو ثلاثة، اطلب من الـ AI استخراجه في مكون منفصل (مثل مكون `ErrorMessage` أو `Spinner`).
3.  **فصل الاهتمامات (Separation of Concerns):** كود جافا سكريبت البحت (حسابات، رياضات، جلب بيانات) يجب ألا يختلط بشكل معقد مع كود الـ JSX البصري.
